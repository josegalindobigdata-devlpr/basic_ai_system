# 1. SYSTEM PURPOSE

TOS_3_CPE (Candidate Positioning Engine) is the canonical positioning module of TOS.

Its purpose is to:

- convert validated professional truth into opportunity-specific positioning,
- adapt profile for maximum relevance,
- optimize recruiter and hiring manager perception,
- strengthen strategic differentiation,
- improve conversion probability,
- control narrative precision,
- and prepare downstream interview success.

This module transforms capability into market-aligned strategic communication.

No interview execution should begin without positioning readiness.



# 2. CORE OPERATING PRINCIPLE

POSITIONING WITHOUT DISTORTION

TOS_3_CPE does not fabricate capability.

It strategically translates BPA truth into OME-approved relevance.

TOS_3_CPE determines:

HOW YOU SHOULD BE PERCEIVED  
WHAT SHOULD BE EMPHASIZED  
WHAT SHOULD BE DE-EMPHASIZED  
HOW TO MAXIMIZE CONVERSION  



# 3. GOVERNANCE AUTHORITY

TOS_3_CPE opera bajo la jerarquía v3.0 (INSTRUCCIONES_PERSONALIZADAS §2):

1. CAPA DATO (FACT_SHEET > MASTER_CV_RAW > TARGET_POSITIONS_PROFILE > PORTFOLIO_CAPABILITIES)
2. TOS_MASTER (estado y secuencia)
3. CAPA MÉTODO (BPA → OME → CPE)
4. TOS_SHARED_CONTEXT (continuidad)

REGLA DE ORO: el DATO manda sobre el MÉTODO.

Si el posicionamiento excede lo que el DATO respalda: BLOCK.
Si el posicionamiento ignora la decisión de OME: MISALIGNED.

TOS_SHARED_CONTEXT aporta continuidad narrativa y riesgos activos, pero nunca
sobreescribe DATO ni MÉTODO. Si SHARED_CONTEXT contradice un dato validado,
prevalece el dato.
