# DISEÑO SISTEMA PERSONAL DE PRODUCTIVIDAD
# ACTÚA COMO:
Consultor Senior de Organización Personal, Productividad Ejecutiva y Gestión del Tiempo especializado en profesionales senior en transición laboral.

# CONTEXTO:
Tengo un profesional con más de 20 años de experiencia que actualmente se encuentra en búsqueda activa de empleo después de 3 meses en desempleo.

El profesional necesita crear un sistema personal de trabajo diario que le permita mantener disciplina, motivación, visibilidad del progreso y foco, evitando que la búsqueda de empleo se convierta en una actividad desestructurada.

# OBJETIVO PRINCIPAL:
Diseñar un sistema operativo personal basado en herramientas digitales que permita gestionar de forma eficiente:

1. Búsqueda de empleo.
2. Actualización y evolución del CV.
3. Preparación de entrevistas.
4. Networking profesional.
5. Actualización de conocimientos técnicos.
6. Certificaciones o formación.
7. Organización diaria y semanal.
8. Seguimiento de objetivos personales.

# SITUACIÓN ACTUAL:
Actualmente la gestión de oportunidades laborales está evolucionando desde Excel hacia Notion.

Notion se utiliza como repositorio principal para:
- Empresas objetivo.
- Vacantes.
- Contactos.
- Estado de candidaturas.
- Información de procesos.
- Histórico.

Se mantiene la posibilidad de exportar información periódicamente a CSV como mecanismo de backup.

El profesional busca una herramienta complementaria, más visual y motivante, orientada al día a día.

# REQUISITOS DEL SISTEMA:
Diseña una solución utilizando Trello como herramienta principal de ejecución diaria.

Trello debe funcionar como:
- Panel visual de control.
- Sistema Kanban personal.
- Gestor de hábitos profesionales.
- Planificador semanal.
- Sistema de motivación mediante progreso visible.

No debe duplicar la información de Notion, sino complementarla.

# REALIZA EL ANÁLISIS EN ESTOS BLOQUES:

## 1. FILOSOFÍA DE ORGANIZACIÓN
Explica cómo debería dividirse la información entre:
- Notion (estrategia, información, histórico, base de datos).
- Trello (acción diaria, ejecución, foco).
- Calendario (bloques de tiempo).
- Otras herramientas si aportan valor.

## 2. DISEÑO DEL TABLERO TRELLO

Define un tablero completo incluyendo:

Nombre recomendado del tablero.

Listas principales:
- Nombre.
- Objetivo.
- Tipo de trabajo que contiene.
- Ejemplos de tarjetas.

Ejemplo de flujo:
Backlog → Esta semana → Hoy → En progreso → Esperando → Completado

Evalúa si este enfoque es correcto o propone uno mejor.

## 3. MODELO DE TARJETAS

Define una estructura estándar para las tarjetas:

- Nombre.
- Descripción.
- Checklist.
- Etiquetas.
- Prioridad.
- Fecha límite.
- Campos personalizados.
- Automatizaciones recomendadas.

Incluye ejemplos reales:
- Adaptar CV para una vacante.
- Preparar entrevista.
- Completar formación.
- Contactar recruiter.
- Revisar mercado.

## 4. SISTEMA DE PLANIFICACIÓN

Diseña una metodología diaria:

Rutina de inicio del día:
- Revisión.
- Priorización.
- Selección de tareas.

Rutina de cierre:
- Evaluación.
- Métricas.
- Preparación del siguiente día.

Diseña también una revisión semanal.

## 5. GESTIÓN DEL TIEMPO

Propón una distribución semanal realista para un profesional senior desempleado.

Incluye bloques aproximados para:

- Búsqueda activa.
- Networking.
- Formación.
- Mejora CV/LinkedIn.
- Preparación entrevistas.
- Descanso y desconexión.

Evita planteamientos de jornada laboral artificial.

## 6. MOTIVACIÓN Y CONTROL

Diseña mecanismos para mantener constancia:

- Métricas visibles.
- Indicadores de progreso.
- Sistema de puntos o logros si aporta valor.
- Seguimiento semanal.

Debe evitar sensación de fracaso si una semana hay menos avances.

## 7. AUTOMATIZACIÓN

Recomienda automatizaciones útiles:

- Butler de Trello.
- Recordatorios.
- Movimiento automático de tarjetas.
- Plantillas.

## 8. HERRAMIENTAS ALTERNATIVAS

Evalúa si existen herramientas más adecuadas o complementarias:

Ejemplos:
- Todoist.
- TickTick.
- ClickUp.
- Asana.
- Obsidian.
- Notion Calendar.
- Google Calendar.

Compara ventajas e inconvenientes.

## 9. PROPUESTA FINAL

Entrega una arquitectura recomendada:

Ejemplo:

Notion:
"Centro de inteligencia profesional"

Trello:
"Centro operativo diario"

Calendario:
"Bloques de ejecución"

Incluye:

- Flujo diario recomendado.
- Flujo semanal recomendado.
- Primeros pasos para implementarlo en 2 horas.

# ESTILO DE RESPUESTA:
Sé práctico, estructurado y orientado a implantación.

No entregues teoría genérica de productividad.
Diseña un sistema realista para un profesional senior en transición laboral.

# REGLAS DE CALIDAD
No proponer dashboards o métricas sin definir previamente la fuente de datos, modelo de captura, reglas de cálculo y mecanismo de actualización. Toda métrica debe tener origen, fórmula y responsable de actualización.
